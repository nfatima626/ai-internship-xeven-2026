## Key Learnings

### 1. Variables
- Variables are names that reference objects stored in memory.
- Python variables do not need an explicit data type declaration because Python is dynamically typed.
- Using descriptive variable names makes code easier to read and maintain.

### 2. Python Data Types
I learned the basic built-in data types:

- `int` for whole numbers
- `float` for decimal numbers
- `bool` for logical values (`True` and `False`)
- `str` for text

I also learned that Python provides other built-in data types such as lists, tuples, dictionaries, and sets.

### 3. Type Checking
- The `type()` function is used to identify the data type of a variable at runtime.
- It helps verify that data is stored in the expected format.

### 4. Type Conversion
I learned how to convert between different data types using built-in functions:

- `int()`
- `float()`
- `str()`
- `bool()`

### 5. Input and Output
- `input()` is used to receive user input.
- `print()` is used to display information to the user.
- I learned how to use f-strings to create clean and readable output.

### 6. Python Memory Management
Through research, I learned that:

- Everything in Python is an object.
- Variables store references to objects rather than the objects themselves.
- Python automatically manages memory using reference counting and garbage collection.

### 7. Mutable vs Immutable Objects
I learned the difference between mutable and immutable data types.

Mutable:
- list
- dictionary
- set

Immutable:
- int
- float
- bool
- string
- tuple

Mutable objects can be modified after creation, while immutable objects cannot.

### 8. PEP 8 Naming Conventions
I learned several professional coding standards:

- Use descriptive variable names.
- Follow `snake_case` for variables and functions.
- Use `UPPER_CASE` for constants.
- Avoid Python keywords as variable names.
- Keep code clean and readable.
